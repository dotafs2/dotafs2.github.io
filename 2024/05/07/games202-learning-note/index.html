# Leature 3

# 如何计算阴影

  1. one pass: 把 light 通过 light 的 mvp，存放所有 light 能看到的点的位置到一张 shadowmap。
  2. two pass: 遍历所有 camera 能看到的点，把这些点和 light 的位置进行连线，如果连线的距离大于第一步得到的 shadowmap 当前坐标的 depth 值，代表 light 没有看到这个点，被遮挡了。如图 2 第二个点所示。

# 为啥存在自遮挡 (self occlusion)，需要加入 bias

出现搓衣板主要是因为在 one pass,light 计算所有可见点的时候，并不是线性的，而是一个片段有一个深度比如 x 到 x+1，这也就造成了在第二个 pass 的时候值总会长一点点。如果光线垂直于平面的时候，这种 bias 会变得很小，反之如果几乎平行与水平面的话 bias 变得很大。

加入 bias 之后，也会有缺点，阴影会丢失一些原本可能造成遮挡的地方。如下图所示，离光源远的地方 bias 比较明显，所以 second pass 的时候减少的 bias 很好的解决了自遮挡，但是离脚近的地方光源角度比较小，bias 所需比较少，所以会造成本来应该存在的阴影被当作是自遮挡，也就消失了。

  • 工业界没人会解决这个问题，但是会找到一个还不错的 bias。学术界有方法能解决:second-depth shadow mapping。
  • 还有另外一个问题 Aliasing (走样)。
