# 5.8

  1. color is generate by distribution of wavelengths.

255 是因为通常 8 字节表示颜色。

  1. subtractive color system (painting,etc..) 颜色越多越黑

  2. binary images: 1 bit per pixel (01 组成的图片), greyscale images: 灰度图 1 byte per pixel.

  3. 一个 byte 表示所有颜色: 3 bit red, 3 bit green, 2 bit blue. 生成 specific 的 256 个 colors。

  4. GIF: 1 byte per pixel. 和上面的不同. each value is an index into a lookup table, 这个 lookup table 有 full RGB values。

  5. 16-bit per pixel: 5 bit red , 6 bit green, 5 bit blue. 65536 个 colors。

  6. RAW images: 48-bit per pixel: 16-bits per color channel , 2^48 colors. high dynamic range images.

  7. 32-bit float per color channel: 96 bit per pixel. internel representation during rendering. RGB range in [0,1].

  8. Image File Format

    • GIF: graphics interchange format
    • JPG: compressed image format natrual images 18-bit/channel
    • TIFF:non-lossy, any data depth
    • PNG: portable network graphis non-lossy 8-bit/16-bit/channel
    • EXR: floating point format 32-bit/16-bit
    • PPM: portable pixel map, no external libraries to read and write. uncompressed format, generally 8 bits per pixel.

  9. 关于作业。存在 read 和 write PPM 的框架。

# Bresenham's line algorithm

两点之间画直线在网格中只有两种情况:

  1. X 的值每次增长 1
  2. Y 的值会在保持不变和增长 1 之间选择

给定两点：起点 (x1, y1), 终点 (x2, y2), 连接他们的理论直线可以用一元一次方程 y=mx+b 来描述，规定 m∈ (0,1) 并且这条线是指向第一象限的线。应用 Bresenham 算法:

定义一个参数 Pi 用于判断下一个小方块的 y 值:
是保持不变还是增加 1

# 泊松盘采样 (Poisson disk)

  1. 从激活点集中随机选出一个点（激活点集是已采样点集的子集）
  2. 在以该点为中心，内半径为 r / 外半径为 2r 的圆环上随机采 k 个点（文中推荐 k=30）
  3. 对 k 个点进行逐一检查，如果某个点与所有已经采样的点距离都小于 r，则将该点放入激活集和采样点集。
  4. 一旦有一个点成功，立即结束该轮检查，如果都没成功，将移除激活点集
